## 回归检查清单

每次提交代码前，对照以下检查项：

### 前端检查
- [ ] 右键粘贴是否仍然可用（terminal-view.ts contextmenu 监听器）
- [ ] 双击复制命令是否仍然可用（main.ts tips-cmd dblclick 监听器）
- [ ] 拖拽文件到终端是否仍然可用（tauri://drag-drop 事件）
- [ ] 拖拽图片到 Notepad 是否仍然可用
- [ ] Ctrl+V 粘贴图片是否仍然可用
- [ ] 主题切换是否正常
- [ ] 快捷键是否正常工作

### 后端检查
- [ ] 创建终端是否正常
- [ ] 写入终端是否正常
- [ ] 剪贴板读写是否正常
- [ ] 文件选择对话框是否正常

### UI 检查
- [ ] 标签页切换是否正常
- [ ] Notepad 发送是否正常
- [ ] 使用技巧面板是否正常打开
- [ ] 历史会话面板是否正常

## 问题报告流程

当用户反馈问题时：

1. **记录问题** - 添加到本清单的「待解决」部分
2. **分析原因** - 确定问题根因
3. **修复问题** - 实施修复方案
4. **移动到已解决** - 填写完整信息（修复日期、修复方案、触发场景）
5. **更新回归检查** - 如需要，添加新的检查项

---

## 文件格式说明

- **ID 格式**: `#001`, `#002` ... 按顺序递增
- **日期格式**: `YYYY-MM-DD`
- **修复方案**: 简短描述修复的代码改动
- **触发场景**: 用户执行什么操作时会遇到这个问题
