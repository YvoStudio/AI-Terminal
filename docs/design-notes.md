# AI-Terminal 设计笔记

## 项目现状（2026-03-10）

基于 Electron + TypeScript + xterm.js + node-pty 的多标签 Claude Code 终端管理器，已实现功能：

- 多标签终端（独立 PTY、状态指示）
- Claude/aider/cursor/copilot 自动检测
- 聊天历史侧边栏
- 笔记板（按标签存储文本块，一键发送）
- 历史工作区（最近 15 个，一键恢复）
- 10 种主题
- 快捷键（Cmd+T/W/1-9/Shift+[/]）
- 文件/图片/目录选择器
- WebGL 加速渲染

> 另有一个 Tauri 跨平台版本正在并行开发中，本文档的设计方案待 Tauri 版完成后合入。

## AI 支持策略

**Claude 优先，其他 AI 兼容为辅：**

- 核心功能（resume 集成、侧边栏历史、idle 检测、自动提交）专为 Claude 深度设计
- 其他 AI（Gemini CLI、Aider、Cursor 等）检测到命令后正常开标签、终端正常使用，不强行适配深度功能
- 后续按需评估是否对某个 AI 做深度集成，能兼容则兼容，不能则不强求

## 待完善功能设计

### 一、笔记板增强 ✅ 已确定方案

**核心场景：** 使用 Claude 过程中等待响应时，快速记录后续要发送的内容（草稿箱）。

#### 标签页设计

| 标签 | 说明 |
|------|------|
| **待发送** | 默认显示，保持主界面简洁 |
| **已发送** | 切换查看历史，按时间倒序排列 |

#### 发送模式（顶部开关切换）

- **手动模式** — 点击"发送"按钮逐条发送
- **自动模式** — 检测到 Claude idle 状态时，自动按顺序发送队列中下一条

#### 笔记块操作

- 拖拽排序（控制发送优先级）
- 每条有"立即发送"按钮（插队场景）
- 发送后自动移入"已发送"标签页

#### 自动提交流程

1. 用户写好笔记块，按顺序排列
2. 检测到 Claude 等待输入（idle 状态）
3. 自动发送队列中第一条
4. 该条移入"已发送"标签页
5. 下次 idle 时继续发送下一条

### 二、`::` 快捷命令面板 ✅ 已确定方案

**触发方式：**
- 单独输入 `::` → 弹出浮层菜单，方向键选择，Enter 确认，Esc 关闭
- 输入 `::关键词` → 直接执行，支持 Tab 补全

**初始菜单：**

| 命令 | 功能 |
|------|------|
| `::resume` | 弹出历史会话列表，选择恢复（新建标签） |
| `::dangerzone` | 危险模式启动 Claude（--dangerously-skip-permissions） |
| `::new` | 新建 Claude 会话（新标签） |
| `::open` | 选择目录在当前标签打开 |
| `::theme` | 切换主题 |

### 六、claude --resume 集成 ✅ 已确定方案

**核心场景：** 恢复之前的 Claude 会话，保留完整上下文。

#### 行为设计

- 每次执行 `claude --resume` 自动新建一个标签
- 标签名从 JSONL 会话文件中读取（会话名/项目名）
- 侧边栏自动加载该会话的历史记录

#### 侧边栏展示

- 用分隔线区分"恢复历史"和"本次会话"
- 默认显示最近 **20 条**历史，超出部分折叠，点击展开全部

```
┌─────────────────┐
│ — 本次会话 —    │
│ • 用户输入...   │
│                 │
│ — 恢复历史 —    │
│ • 之前的输入... │  ← 默认显示 20 条
│ [展开更多...]   │  ← 超出折叠
└─────────────────┘
```

#### 历史工作区联动

- 历史工作区每条记录额外存储对应的 **session ID**
- 从历史工作区恢复时：
  1. 新建标签
  2. 自动在该目录执行 `claude --resume <session-id>`
  3. 侧边栏加载对应历史
- 历史工作区从"只恢复目录"升级为"恢复完整上下文"

### 七、通知系统 ✅ 已确定方案

**触发条件：** 始终触发（无论窗口是否在前台）

**触发事件：**
- Claude 任务完成（idle，等待输入）
- 报错
- 等待用户输入（如权限确认）

**通知方式：**
- Windows：任务栏图标闪烁
- macOS：Dock 图标显示红点 + 跳动

**红点数字：** 显示有待处理的标签数（等待输入/需要确认的标签数量）
