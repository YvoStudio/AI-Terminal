# 问题追踪与回归检查技能

## 技能名称
`bug-tracker`

## 触发命令
`/bug-tracker` 或 `/bugs`

## 功能说明
收集用户反馈的问题，维护问题清单，在代码修改后自动检查是否引入了已修复问题的回归。

## 使用方式

### 1. 报告新问题
```
/bug-tracker report <问题描述>
```
示例：
```
/bug-tracker report 右键粘贴终端没反应
```

### 2. 查看问题清单
```
/bug-tracker list
```
显示所有已记录的问题及其状态（待解决/已解决/已归档）。

### 3. 标记问题为已解决
```
/bug-tracker resolve <ID> <修复方案>
```
示例：
```
/bug-tracker resolve #001 添加 contextmenu 事件监听器
```

### 4. 回归检查（每次修改代码后自动执行）
```
/bug-tracker check
```
检查代码修改是否重新引入了已修复的问题。

---

## 工作流程

### 当用户报告问题时：

1. **读取问题清单** - 读取 `docs/问题追踪清单.md`
2. **分配 ID** - 下一个可用 ID（如 #010）
3. **添加到待解决** - 填写问题描述、反馈日期、优先级
4. **确认记录** - 告知用户问题已记录，显示 ID

### 当修复问题时：

1. **实施修复** - 编写代码修复问题
2. **测试验证** - 确保修复有效
3. **更新清单** - 移动到「已解决」部分，填写：
   - 修复日期
   - 修复方案（代码改动简述）
   - 触发场景（什么操作会触发这个问题）
4. **提交说明** - commit 信息中包含问题 ID

### 当开发新功能时：

1. **修改前检查** - 读取「回归检查清单」
2. **修改后验证** - 对照检查清单逐项验证
3. **发现问题** - 如发现回归，立即记录并优先修复

## 回归检查清单（快速参考）

### 核心功能检查
| 功能 | 测试方法 |
|------|----------|
| 右键粘贴 | 右击终端区域，应粘贴剪贴板内容 |
| 双击复制命令 | 打开使用技巧面板，双击 AI 命令，应显示「已复制」 |
| 拖拽文件到终端 | 拖拽文件到终端，应输入文件路径 |
| 拖拽图片到 Notepad | 拖拽图片到笔记板，应显示图片预览 |
| Ctrl+V 粘贴 | 复制文本后 Ctrl+V，应粘贴到终端 |
| 标签切换 | 点击标签页，应正确切换 |
| Notepad 发送 | 点击发送按钮，文本应发送到终端 |

### 提交前必须确认
- [ ] 问题清单已更新（如有新修复）
- [ ] 回归检查已通过
- [ ] 没有重新引入已知问题

## 自动化建议

如果未来添加测试框架，应为此清单中的每个问题编写回归测试用例。

---

## 技能维护

- 定期（每月）审查「待解决」问题，评估优先级
- 已修复超过 3 个月未复现的问题可移入「历史归档」
- 如发现重复出现的问题，说明修复不彻底，需重新分析根因
